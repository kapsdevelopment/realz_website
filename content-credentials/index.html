Content Credentials for verified photo exports

Realz is capture-first: the proof starts when a photo is taken inside the app. C2PA-compatible Content Credentials add a second layer for exported files, helping provenance metadata travel with the image.

Best fit

Verified photo exports for publishing, documentation, review, and professional workflows where the image file itself should carry a readable provenance signal.

How C2PA fits the Realz proof model

Realz creates a proof around the captured photo bytes and exposes a public verification page when the user chooses to share. For exported images, Realz can also embed C2PA-compatible Content Credentials that reference the Realz proof and include integrity metadata inside the JPEG file.

What Realz can include

  • A Realz proof URL that people can open independently.
  • The Realz proof identifier for the verified capture.
  • Integrity metadata that connects the export back to the Realz proof.
  • Creator or copyright fields when the user chooses to configure them.

Why this is useful

C2PA-compatible Content Credentials make Realz exports easier to inspect in tools that understand image provenance metadata. This is especially useful when a file moves through publishing, reporting, documentation, or client-review workflows where a plain image file can lose context.

What remains canonical

The Realz verification page remains the canonical proof surface. Image metadata can be stripped by some social platforms, messaging apps, or image pipelines. The verify URL gives people a separate place to check proof state even when metadata does not survive.

What Realz does not claim

Content Credentials do not prove every fact inside a scene, and they do not turn an old imported file into a Realz capture. They help carry provenance information for Realz-verified exports while the underlying Realz proof explains the capture and verification path.

Questions people ask

Is Realz currently claiming C2PA certification?

Realz describes the feature as C2PA-compatible Content Credentials. We avoid claiming C2PA certification, conformance, or trusted-provider status unless that process has been formally completed.

Does C2PA slow down taking the photo?

The intended Realz model is to generate Content Credentials during export or sharing, not during the shutter moment. Capture proof remains the early security layer; C2PA is most valuable when a verified file is distributed.

Realz layers

  • Capture in the Realz app.
  • Create a signed proof for the captured bytes.
  • Share a public Realz verify URL.
  • Optionally export a JPEG with C2PA-compatible Content Credentials.
